# ISSUE-001: 모니터 페이지 진입 시 응시 중 수험자가 없으면 이후 진입한 수험자를 자동 감지하지 못함

- **심각도**: Low
- **관련 TC**: TC-003
- **재현 경로**:
  1. 관리자가 모니터 페이지에서 시험을 선택 (현재 응시 중인 수험자 없음)
  2. 수험자가 해당 시험에 새로 진입 (ExamSession 생성)
  3. 모니터 페이지에서 자동 갱신을 기다림
  4. 수험자가 나타나지 않음 (폴링이 시작되지 않았으므로)
- **기대 동작**: 수험자가 새로 진입하면 폴링 또는 다른 메커니즘으로 자동 감지되어야 함
- **실제 동작**: `startPollingIfNeeded()`가 현재 `IN_PROGRESS` 상태인 세션이 있을 때만 폴링을 시작하므로, 초기 로드 시 응시 중 수험자가 없으면 폴링 자체가 시작되지 않음. 수동으로 드롭다운을 재선택하면 데이터가 갱신됨.
- **환경**: Chromium, http://localhost:5173

## 상세 설명

`ExamMonitor.vue`의 `startPollingIfNeeded()` 함수는 `sessions.value.some(s => s.status === 'IN_PROGRESS')`가 true일 때만 10초 간격 폴링을 시작한다. 초기 로드 시 모든 수험자가 "제출 완료" 또는 "시간 만료"이거나 수험자가 아예 없는 경우, 이후 새로운 수험자가 진입해도 폴링이 시작되지 않아 관리자가 수동으로 시험을 재선택해야 한다.

## 제안 수정 방향

1. **최소한의 주기적 폴링**: IN_PROGRESS 수험자가 없어도 30~60초 간격으로 느린 폴링을 유지하여 새로운 진입을 감지
2. **SSE 연동**: 수험자 진입 시 기존 SSE 알림 시스템을 활용하여 관리자 모니터 페이지에 즉시 알림
3. **현재 상태로 두되 UI 안내**: "새로고침하려면 시험을 다시 선택하세요" 안내 표시

**참고**: 일반적인 사용 시나리오에서는 관리자가 수험자 진입 전에 모니터 페이지를 열어두는 경우가 드물고, 수험자가 이미 진입한 상태에서 모니터링하는 것이 주 사용 패턴이므로 심각도는 Low로 판정한다.
