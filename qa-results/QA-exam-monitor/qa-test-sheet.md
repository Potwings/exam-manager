# QA Test Sheet: 응시 현황 모니터링 페이지

- **테스트 일시**: 2026-03-05
- **테스트 대상**: 관리자가 현재 시험 응시 중인 수험자 목록과 남은 시간을 실시간 모니터링하는 페이지
- **테스트 환경**: Chromium (Playwright MCP), Frontend http://localhost:5173, Backend http://localhost:8080

## 테스트 케이스

### TC-001: 페이지 접근 및 헤더 "응시현황" 탭 확인
- **분류**: 기능
- **우선순위**: P0-Critical
- **사전 조건**: 관리자 계정 (admin/admin123) 존재, 서버 기동 상태
- **테스트 단계**:
  1. http://localhost:5173/admin/login 접근
  2. admin / admin123 로그인
  3. 헤더 네비게이션 탭 순서 확인
  4. "응시현황" 탭 클릭하여 /admin/monitor 이동 확인
  5. 페이지 제목과 설명 텍스트 확인
- **기대 결과**: 헤더에 "응시현황" 탭이 "채점결과"와 "시험관리" 사이에 위치. 클릭 시 /admin/monitor 페이지로 이동. 제목 "응시 현황", 설명 "현재 시험 응시 상태를 모니터링합니다."
- **실제 결과**: 로그인 성공 후 /admin/scores 리다이렉트. 헤더 탭 순서: 채점결과 | 응시현황 | 시험관리 | 계정관리. "응시현황" 클릭 시 /admin/monitor 이동. 제목/설명 정확히 표시됨.
- **상태**: PASS
- **비고**: 없음

### TC-002: 시험 드롭다운 표시 및 활성 시험 자동 선택
- **분류**: 기능
- **우선순위**: P0-Critical
- **사전 조건**: 관리자 로그인 완료, /admin/monitor 페이지
- **테스트 단계**:
  1. /admin/monitor 페이지에서 시험 드롭다운 확인
  2. 드롭다운에 시험 목록이 "시험명(N문제)" 형식으로 표시되는지 확인
  3. 활성(active) 시험이 자동 선택되어 있는지 확인
  4. 다른 시험으로 변경하여 데이터 갱신 확인
- **기대 결과**: 시험 목록이 드롭다운에 표시되고, 활성 시험이 자동 선택됨. 시험 변경 시 데이터 갱신됨.
- **실제 결과**: 3개 시험 표시 ("크리니티 채용 적성검사 (Java 기반 Web 개발) (14문제)", "Test Exam (1문제)", "PR Test Exam (2문제)"). 활성 시험 "PR Test Exam (2문제)"가 자동 선택됨. "Test Exam" 선택 시 "응시자가 없습니다" 표시, 다시 "PR Test Exam" 선택 시 응시자 테이블 갱신됨.
- **상태**: PASS
- **비고**: 없음

### TC-003: 수험자 시험 진입 후 "응시 중" 상태 표시
- **분류**: 기능
- **우선순위**: P0-Critical
- **사전 조건**: 활성 시험 존재, 관리자 모니터 페이지 열려 있음
- **테스트 단계**:
  1. 새 탭에서 수험자 로그인 (이름: "QA Monitor Test", 생년월일: "19950315")
  2. "시험 시작" 클릭하여 시험 진입
  3. 관리자 모니터 페이지에서 드롭다운 재선택으로 데이터 갱신
  4. "응시 중" Badge 표시 확인
  5. 이름, 생년월일, 남은 시간, 시작 시각 확인
- **기대 결과**: 수험자가 테이블에 "응시 중" Badge로 표시되고, 남은 시간이 MM:SS 형식으로 표시됨
- **실제 결과**: "QA Monitor Test" 수험자가 "응시 중" Badge(기본 variant, 검은색)로 표시. 생년월일 1995-03-15, 남은 시간 29:35 (MM:SS), 시작 시각 08:53:11.
- **상태**: PASS
- **비고**: 폴링이 시작되지 않은 상태에서는 드롭다운 재선택으로 수동 갱신 필요

### TC-004: 남은 시간 MM:SS 형식 및 제출 완료 수험자 "-" 표시
- **분류**: 기능
- **우선순위**: P1-High
- **사전 조건**: TC-003 완료 (응시 중 수험자 + 제출 완료 수험자 혼재)
- **테스트 단계**:
  1. 응시 중 수험자의 남은 시간 컬럼이 MM:SS 형식인지 확인
  2. 제출 완료 수험자의 남은 시간이 "-"인지 확인
- **기대 결과**: 응시 중 수험자는 MM:SS, 제출 완료 수험자는 "-"
- **실제 결과**: "QA Monitor Test" (응시 중): 29:35 (MM:SS 형식). "QA Tester" (제출 완료): "-". "Test User" (제출 완료): "-".
- **상태**: PASS
- **비고**: 없음

### TC-005: 수험자 답안 제출 후 "제출 완료" 상태 변경
- **분류**: 기능
- **우선순위**: P0-Critical
- **사전 조건**: TC-003에서 수험자가 응시 중 상태
- **테스트 단계**:
  1. 수험자 탭에서 Q1에 답안 입력
  2. "답안 제출" 클릭 -> 확인 다이얼로그에서 "제출" 클릭
  3. 관리자 모니터 페이지에서 폴링 대기 (약 11초)
  4. "제출 완료" Badge로 변경 확인
  5. 남은 시간이 "-"로 변경 확인
- **기대 결과**: 수험자 상태가 "응시 중" -> "제출 완료"로 변경, 남은 시간 "-"
- **실제 결과**: 수험자 답안 제출 성공. 약 11초 후 폴링으로 "QA Monitor Test"가 "제출 완료" Badge(secondary variant, 회색)로 자동 변경됨. 남은 시간 "-"로 정상 표시.
- **상태**: PASS
- **비고**: 폴링이 IN_PROGRESS 수험자가 있을 때만 동작하여 자동 갱신됨

### TC-006: 하단 요약 섹션 카운트 정확성
- **분류**: UI/UX
- **우선순위**: P1-High
- **사전 조건**: 응시자 데이터 존재
- **테스트 단계**:
  1. 모니터 페이지 하단 요약 텍스트 확인
  2. 총 인원, 응시 중, 제출 완료 수치가 테이블 데이터와 일치하는지 확인
- **기대 결과**: 요약 섹션의 카운트가 테이블 데이터와 정확히 일치
- **실제 결과**: TC-003 시점: "총 3명 . 응시 중 1명 . 제출 완료 2명" (테이블: 응시중 1 + 제출완료 2 = 3 일치). TC-005 이후: "총 3명 . 응시 중 0명 . 제출 완료 3명" (테이블: 제출완료 3 = 3 일치).
- **상태**: PASS
- **비고**: 없음
