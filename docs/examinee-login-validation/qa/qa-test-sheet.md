### TC-004: 재시험 방지 - 이미 응시 완료 수험자 로그인 차단
- **분류**: 기능
- **우선순위**: P0-Critical
- **사전 조건**: 활성 시험 존재, 해당 시험에 제출 기록이 있는 수험자
- **테스트 단계**:
  1. `/exam/login` 페이지 접속
  2. 이름: "테스트유저", 생년월일: "20000101" 입력 (TC-001에서 제출 완료)
  3. [시험 시작] 버튼 클릭
- **기대 결과**: 페이지 이동 없이 "이미 응시 완료한 시험입니다." 에러 메시지 표시 (409 CONFLICT)
- **실제 결과**: 로그인 API가 200 OK 반환, `/exam/take/11`로 이동됨 (재시험 방지 미동작)
- **상태**: FAIL
- **비고**: ISSUE-001 참조. 백엔드 ExamineeService의 `existsByExamineeIdAndProblemExamActiveTrueAndProblemExamDeletedFalse` 쿼리 메서드가 false를 반환하는 것으로 추정. 홍길동(기존 제출자)으로도 200 OK 확인.
